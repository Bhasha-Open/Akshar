Command Line Interface#

akshar provides command-line tools for tokenization, analysis, and model training.

Basic Usage#

All commands follow this pattern:

akshar <command> [options] <input>

Available commands:

  • tokenize - Convert text to tokens

  • detokenize - Convert tokens back to text

  • explain - Detailed analysis

  • train - Train tokenization model

tokenize#

Tokenize input text:

akshar tokenize "आज मौसम अच्छा है"

Output:

['आ', 'ज', ' ', 'मौ', 'स', 'म', ' ', 'अ', 'च्', 'छ', 'ा', ' ', 'है']

Options:

--model PATH           # use trained model
--model-type TYPE      # sentencepiece|bpe
--format text|json|id  # output format (id requires --model)
-i, --input FILE       # read text from file
-o, --output FILE      # write output to file

detokenize#

akshar detokenize "▁aaj ▁मौ सम ▁बहुत ▁अच्छा ▁hai" --model models/akshar.model

explain#

akshar explain "क्षेत्रे धर्मक्षेत्रे समवेता युयुत्सवः"

train#

SentencePiece:

akshar train data/corpus.txt --output models/akshar --vocab-size 24000

HuggingFace BPE:

akshar train data/corpus.txt --output models/akshar --model-type bpe --vocab-size 32000

Advanced Usage#

  • Batch process files via shell loops.

  • Use --no-preprocess if your corpus is already cleaned.

  • For IDs, always supply --model.

API#

Command-line interface for Akshar.

Commands#

tokenize:

Tokenize text from an argument or file, with optional model and formats.

detokenize:

Convert tokens back to text (best-effort heuristics per model type).

explain:

Print a human-readable breakdown of the processing pipeline.

train:

Train a SentencePiece or BPE tokenizer from a corpus.

detokenize_command(args)

Handle the ‘detokenize’ subcommand.

Parameters:

args – argparse.Namespace with fields: - tokens (str | None) space-separated tokens if not using –input - input (str | None) file path containing tokens or JSON list - output (str | None) destination path - model (str | None), model_type (str) for model-aware heuristics

explain_command(args)

Handle ‘explain’ - print a detailed pipeline breakdown.

Shows normalized text, akshars, script segments, tokens, and stats.

main()

Main CLI entry point. Dispatches to subcommands.

preprocess_corpus(input_file, output_file)

Preprocess a corpus file with Akshar normalization.

Returns:

Path to the written output_file.

Return type:

str

tokenize_command(args)

Handle the ‘tokenize’ subcommand.

Parameters:

args – argparse.Namespace with fields: - text (str | None) - input (str | None) path to a text file - model (str | None) model path if using SP/BPE - model_type (str) ‘sentencepiece’ or ‘bpe’ - format (str) ‘text’ | ‘json’ | ‘id’ - output (str | None) file to write output

train_command(args)

Handle ‘train’ - train SentencePiece or BPE on a corpus.

Parameters:

args – argparse.Namespace containing corpus path, output prefix, and options.
